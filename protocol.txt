STRUKTURA WIADOMOŚCI:
<size_of_json> + 'x' + JSON

Do json dołączone są krotki:
token - losowy numer potwierdzający autoryzację - zawsze po udanym logowaniu
error - w razie problemów dołączony kod błędu


Struktura JSON:

Ze strony klienta:
logowanie:
type: "auth"
username: username
password: password

pobierz listę dostępnych aukcji:
type: "list"
data: dload_trades
who: user

Podbij cenę:
type: "set_price"
price: price


Ze strony serwera:
Autoryzacja logowania:
type: "auth"
token: token (nie istnieje, jeśli logowanie nieudane)

Wyślij listę dostępnych aukcji:
type: "list"
list: [{lista}]

Informacja zwrotna przebicia ceny:
type: "set_price"
answer: "too_low"/"trade_ended"/"accept"
